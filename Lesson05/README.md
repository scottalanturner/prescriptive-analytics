# Lesson 5: Validation and Sensitivity Analysis

Welcome to Lesson 5 of Prescriptive Analytics! This lesson focuses on validating optimization solutions and understanding their sensitivity to assumptions and parameters.

## Learning Objectives

By the end of this lesson, you will understand:

- **Sensitivity Analysis**: How to test how solutions change with parameter changes
- **What-If Analysis**: How to explore different scenarios and assumptions
- **Extreme Value Testing**: How to test solutions under extreme conditions
- **Fragile vs Robust Decisions**: How to identify decisions that are sensitive to assumptions
- **Assumption Stress Testing**: How to test solutions when assumptions are violated
- **Sanity Checks**: How to validate that solutions make practical sense
- **Comprehensive Validation Workflow**: A systematic approach to validating solutions

## Core Concepts

### 1. Sensitivity Analysis
- Testing how solutions change with parameter changes
- Identifying which parameters have the most impact
- Understanding solution stability and reliability

### 2. What-If Analysis
- Exploring different scenarios and assumptions
- Understanding how changes affect recommendations
- Preparing for different possible futures

### 3. Extreme Value Testing
- Testing solutions under extreme conditions
- Identifying when solutions break down
- Understanding solution limits and boundaries

### 4. Fragile vs Robust Decisions
- **Fragile**: Decisions that are very sensitive to assumptions
- **Robust**: Decisions that work well across many scenarios
- Preferring robust decisions when uncertainty is high

### 5. Assumption Stress Testing
- Testing solutions when assumptions are violated
- Understanding how robust solutions are to assumption changes
- Identifying critical assumptions

### 6. Sanity Checks
- Validating that solutions make practical sense
- Checking for obvious errors or unrealistic results
- Ensuring solutions are implementable

### 7. Comprehensive Validation Workflow
- A systematic approach to validating solutions
- Combining multiple validation techniques
- Building confidence in recommendations
